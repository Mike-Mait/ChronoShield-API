DST Gap Detection

Catches invalid times that don't exist during spring-forward transitions. Returns the next valid time automatically.

Overlap Resolution

Detects ambiguous fall-back times and resolves them with your preferred policy — earlier, later, or reject.

See it in action

Three endpoints. Zero ambiguity.

Request 
$ curl -X POST https://api.chronoguard.dev/v1/datetime/validate \
  -H "x-api-key: YOUR_KEY" \
  -d '{
    "local_datetime": "2026-03-08T02:30:00",
    "time_zone": "America/New_York"
  }'
Response status: invalid 
{
  "status": "invalid",
  "reason_code": "DST_GAP",
  "message": "This time does not exist due to DST transition.",
  "suggested_fixes": [
    { "strategy": "next_valid_time", "local_datetime": "2026-03-08T03:00:00" }
  ]
}
Request 
$ curl -X POST https://api.chronoguard.dev/v1/datetime/resolve \
  -H "x-api-key: YOUR_KEY" \
  -d '{
    "local_datetime": "2026-11-01T01:30:00",
    "time_zone": "America/New_York",
    "resolution_policy": { "ambiguous": "earlier" }
  }'
Response resolved 
{
  "instant_utc": "2026-11-01T05:30:00.000Z",
  "offset": "-04:00"
}
Request 
$ curl -X POST https://api.chronoguard.dev/v1/datetime/convert \
  -H "x-api-key: YOUR_KEY" \
  -d '{
    "instant_utc": "2026-06-15T15:00:00Z",
    "target_time_zone": "Europe/London"
  }'
Response converted 
{
  "local_datetime": "2026-06-15T16:00:00",
  "offset": "+01:00",
  "time_zone": "Europe/London"
}
