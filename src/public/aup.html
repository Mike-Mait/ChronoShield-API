Acceptable Use Policy

Last updated: April 7, 2026

1. Overview

This Acceptable Use Policy ("AUP") supplements the Terms of Service and governs acceptable behavior when using the ChronoShield API, website, documentation, and related services (collectively, the "Service").

By using the Service, you agree to comply with this AUP. We reserve the right to update this policy at any time. Material changes will be communicated via the email associated with your API key.

2. Prohibited Uses

You must not use the Service to:

  • Disrupt or degrade the Service — Including denial-of-service attacks, intentional overloading, or any activity designed to impair service availability for other users.
  • Circumvent access controls — Including bypassing API key authentication, rate limits, or other security mechanisms through any means.
  • Scrape or harvest data — Using automated means to systematically extract data from the Service beyond its intended API interface.
  • Facilitate unlawful activity — Using the Service in connection with any activity that violates applicable laws, regulations, or third-party rights.
  • Impersonate or misrepresent — Falsifying your identity, email address, or organizational affiliation when registering for or using the Service.
  • Redistribute API access — Reselling, sublicensing, or providing proxy access to the API without prior written consent.
  • Exploit vulnerabilities — Attempting to probe, scan, or exploit security vulnerabilities in the Service. If you discover a vulnerability, please report it responsibly via our security disclosure process.

3. Rate Limit Abuse

The following activities are considered rate limit abuse:

  • Creating multiple accounts or API keys to circumvent per-key rate limits
  • Rotating API keys programmatically to evade usage tracking
  • Sending requests at a sustained rate designed to consume disproportionate resources
  • Using the batch endpoint to circumvent per-request rate counting

If you need higher limits than your current tier provides, please contact us to discuss an upgrade or enterprise plan.

4. Security Responsibilities

You are responsible for:

  • Keeping your API key confidential and not exposing it in client-side code, public repositories, or logs
  • Rotating your API key immediately if you suspect it has been compromised
  • Implementing appropriate security measures in your own applications that integrate with the API
  • Notifying us promptly at support@chronoshieldapi.com if you become aware of any unauthorized use of your key

5. Monitoring & Enforcement

We monitor API usage patterns to detect abuse, ensure fair resource allocation, and maintain service quality. This monitoring includes request volumes, error rates, and traffic patterns. We do not inspect the content of your API requests beyond what is necessary for processing.

6. Reporting Violations

If you become aware of any use of the Service that violates this AUP, please report it to support@chronoshieldapi.com. We investigate all reports and take appropriate action.

7. Consequences of Violation

Violations of this AUP may result in one or more of the following, at our sole discretion:

  • A written warning
  • Temporary throttling or rate limiting of your API key
  • Temporary suspension of API access
  • Permanent revocation of your API key and termination of your account
  • Referral to law enforcement if criminal activity is suspected

Where possible, we will provide notice before taking action. However, we reserve the right to act immediately and without notice when necessary to protect the Service, its infrastructure, or other users.

8. Contact

Questions about this Acceptable Use Policy can be directed to support@chronoshieldapi.com.
